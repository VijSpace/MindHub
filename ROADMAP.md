# ROADMAP

---

## Overview

This roadmap defines the development phases and boundaries of the project.

The goal is to ensure:

* controlled scope
* clear iteration
* no over-engineering
* alignment with product philosophy

---

## Current Stage

MVP (Phase 1–3)

Focus:

* usability
* clarity
* stable structure

Not focus:

* scaling
* monetization
* growth hacks

---

## Phase 1 — Community (Expression)

### Goal

Enable users to express themselves.

---

### Features

* user authentication
* create post (text only)
* comment system
* basic feed (latest first)

---

### Non-goals

* likes system (optional later)
* recommendation algorithm
* notifications
* social graph

---

### Success Criteria

* user can create a post
* user can comment
* basic interaction exists

---

## Phase 2 — Practice (Action)

### Goal

Help users move from thinking to doing.

---

### Features

* breathing exercise (guided)
* meditation timer
* audio meditation (3–5 items)

---

### Non-goals

* complex tracking analytics
* AI-guided meditation
* personalized recommendation

---

### Success Criteria

* user completes at least one practice
* practice experience is smooth and calm

---

## Phase 3 — Journal & Check-in (Habit)

### Goal

Build user habit and retention.

---

### Features

* daily journal (guided prompts)
* daily check-in button
* streak calculation

---

### Non-goals

* AI-generated journaling
* mood prediction systems
* gamified reward systems

---

### Success Criteria

* user writes at least one journal entry
* streak system works correctly
* user returns next day

---

## Phase 4 — Refinement (Ongoing)

### Goal

Improve product quality and depth.

---

### Areas

* UI/UX refinement
* content expansion
* performance improvement
* small feature enhancements

---

## Product Boundaries (Global)

At all phases:

Do NOT introduce:

* addictive mechanics
* aggressive engagement features
* AI replacing user thinking
* unnecessary complexity

---

## Guiding Principle

Do not build more.

Build clearer.
