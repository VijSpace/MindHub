# Project Context

## Mission

This project helps users stay clear-minded and internally grounded in the AI era.

---

## Product Identity

This is NOT a generic forum.

It is:

* a calm community
* a mindfulness practice tool
* a reflection system

The goal is to support human awareness, not replace it.

---

## Core Modules

* Community (posts + comments)
* Practice (breathing, meditation, timer)
* Journal (daily reflection)
* Check-in (habit tracking)

---

## Current Stage

MVP (Phase 1: Community)

---

## MVP Includes

* simple post creation (text only)
* comment system
* email magic-link login
* nickname editing
* basic community feed

---

## MVP Excludes

* Practice
* Journal
* Check-in
* streak tracking
* recommendation algorithms
* social graph (follow / DM)
* notifications system
* complex AI agents
* AI writing user content

---

## Product Philosophy

* users should think and feel, not outsource everything to AI
* the system should reduce anxiety, not increase it
* the system should not be addictive

---

## Design Tone

* calm
* minimal
* spacious
* quiet

Avoid:

* bright colors
* noisy UI
* aggressive engagement patterns

---

## AI Principle

AI should:

* assist reflection
* help summarize
* support understanding

AI should NOT:

* replace journaling
* generate fake emotions
* dominate user decisions

---

## Engineering Principles

* code must be readable and modular
* structure should be domain-based
* avoid over-engineering
* prioritize long-term maintainability

---

## Naming Conventions

Use consistent terms:

* Community (not forum/feed mixed)
* Practice (not tools/meditation/random)
* Journal (not diary/log mixed)
* Check-in (not checkin/sign-in/multiple variants)

---

## Before Making Changes

Always read:

1. README.md
2. PROJECT_CONTEXT.md
3. ROADMAP.md
4. relevant docs in /docs

---

## Goal for Contributors / AI Models

Make the system:

* easier to understand
* easier to extend
* more aligned with product philosophy

NOT:

* more complex
* more feature-heavy
* more distracting
