# Workspace Rules

This file defines the top-level collaboration rules for the `D:\Mindhub` workspace.

## Read First

Before making product or code changes, read:

1. `README.md`
2. `README.zh.md`
3. `PROJECT_CONTEXT.md`
4. `ROADMAP.md`
5. `app/README.md` when running or configuring the app
6. `app/AGENTS.md` when editing the Next.js app

## Responsibility Split

Top-level docs in `D:\Mindhub` describe:

- product identity
- roadmap and scope
- naming
- collaboration rules

Docs inside `D:\Mindhub\app` describe:

- app setup
- local development
- environment configuration
- framework-specific coding constraints

The actual git repository is `D:\Mindhub\app`.

## Product Guardrails

Keep the product:

- calm
- minimal
- quiet
- readable
- maintainable

Do not add:

- addictive loops
- noisy engagement mechanics
- unnecessary complexity
- AI that replaces user reflection

Current delivery focus remains Phase 1: Community.
Do not start formal Practice / Journal / Check-in features yet.

## Naming

Use consistent domain terms:

- Community
- Practice
- Journal
- Check-in
